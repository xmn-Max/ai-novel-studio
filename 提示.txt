# AI 小说转剧本工具 — 完整详细流程

## 1️⃣ 输入阶段

**用户提供**

* 小说文本（3 章以上）
* 小说类别（武侠 / 玄幻 / 科幻 / 言情 / 叙事 / 魔幻）(界面有这六个选项)

**系统操作**

1. 记录小说类别，决定后续 YAML 的生成规则（场景、任务、角色等权重）

   * 武侠：动作/冲突优先，英雄侠客为主角候选
   * 玄幻：修炼/冒险优先，修炼者/天才为主角候选
   * 科幻：科技/探索优先，科学家/探险者为主角候选
   * 言情：情感/人物关系优先，情侣/主要情感人物为主角候选
   * 叙事：故事结构/心理描写为主，普通人物或主角为候选
   * 魔幻：魔法/奇异事件优先，魔法师/英雄为主角候选

2. 对原文进行文本清洗（去掉空格、多余换行、特殊符号），方便后续统计

---

## 2️⃣ 小说解析阶段（Novel Parser）

**目标**：结构化小说内容，为角色提取和剧情分析提供基础数据

**操作细节**

1. 拆分章节
2. 统计每章字数
3. 保留章节标题与原文段落
4. 生成 `novel` 数据结构

```yaml
novel:
  title: "小说标题"
  chapters:
    - title: "第一章 xxx"
      text: "原文内容"
    - title: "第二章 xxx"
      text: "原文内容"
  word_count: 25000
```

---

## 3️⃣ 角色提取阶段（Character Extractor）

**目标**：识别小说中所有重要角色，生成角色清单

**操作细节**

1. 从小说文本中提取人名、代号
2. 使用 NLP/正则识别性别、年龄、典型特征（可选）
3. 建立角色关系（如师兄妹、朋友、情侣）
4. 输出 YAML 角色列表

```yaml
characters:
  - id: C001
    name: 林凡
    gender: 男
    age: 18
    traits: [冷静, 坚韧]
    relationships: [{target: 苏月, relation: 师妹}]
```

---

## 4️⃣ 剧情分析阶段（Plot Analyzer）

**目标**：提取主线剧情、冲突、高潮事件

**操作细节**

1. 对章节进行主题/情节分类（主线/支线）
2. 按事件生成 `plot.events` 列表
3. 生成主题、冲突和高潮节点

```yaml
plot:
  theme: "江湖恩怨与侠义成长"
  conflict: "林凡被迫参加门派考核"
  climax: "最终比武夺冠"
  ending: "林凡成为新一代门派掌门"
  events:
    - id: E001
      description: "林凡进门派"
    - id: E002
      description: "遭遇试炼"
```

---

## 5️⃣ 场景规划阶段（Scene Planner）

**目标**：将剧情事件映射为具体剧本场景

**操作细节**

1. 遍历 `plot.events`
2. 为每个事件生成场景 ID、地点、时间
3. 列出场景中涉及的角色
4. 为动作和对白做初步规划

```yaml
scene_plan:
  - scene_id: S001
    purpose: "主角进入宗门"
    location: "山门"
    time: "白天"
    event_refs: [E001]
```

---

## 6️⃣ YAML 剧本生成（Script Generator）

**目标**：根据 Scene Plan 生成完整 YAML 剧本

**操作细节**

1. 将场景映射为 `script.scenes`
2. 填充动作 (`action`) 和对白 (`dialogue`)
3. 每个对白引用角色 ID，动作保持简洁

```yaml
script:
  scenes:
    - scene_id: S001
      heading:
        location: "山门"
        time: "白天"
      characters: [C001, C002]
      action: ["林凡来到山门参加考核"]
      dialogue:
        - speaker: C001
          text: "我一定要通过考核"
        - speaker: C002
          text: "那可不容易"
```

---

## 7️⃣ 主角验证 & Count 流程（核心验证逻辑）

**目标**：验证 YAML 是否与原文一致

**流程详细步骤**

1. **初始化计数器**

```python
count = 0
```

2. **Step 1：确定 YAML 主角**

* 遍历 YAML `script.scenes` 的 `characters` 和 `dialogue.speaker`
* 统计每个角色的出现次数
* 根据小说类型规则（武侠 → 第一个侠客，玄幻 → 修炼者等）挑选 `main_character_candidate`
* 统计该角色在 YAML 中主要任务出现次数 → 记录 `main_tasks`

3. **Step 2：原文频率验证**

* 遍历每章原文
* 统计主角出现次数（名字 + 别名 + 代词 he/she/it/他/她/它）
* 每章出现次数 ≥ 5 → 通过

```python
if all(chapter_count >= 5 for chapter_count in main_character_appearances):
    count += 1
```

4. **Step 3：任务一致性验证**

* 遍历 YAML 中主角的动作 (`main_tasks`)
* 判断原文中相关章节中涉及的事件是否与这些动作有关
* 若大于一半事件相关 → count += 1

5. **Step 4：重试机制**

* 若 count = 1 → 重新扫描 YAML 及原文（可调节策略）
* 若仍 count = 1 → 直接返回结果

6. **输出 Count 及验证状态**

```yaml
validation:
  main_character: 林凡
  count: 2
  status: "验证通过"
```

---

## 8️⃣ Schema 校验阶段（Schema Validator）

**目标**：保证生成 YAML 满足 Schema

**操作细节**

1. 验证每个场景：

* `scene_id` 不为空
* `characters` 必须引用已存在角色 ID
* `dialogue.speaker` 必须在 `characters` 列表中
* `location` 不为空

2. 若校验失败 → 自动修复或提示用户
3. 校验通过 → 写入 Project 中
